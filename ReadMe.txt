Постановка задачи: создать консольное приложение для перемножения матриц, сделать самостоятельную реализацию алгоритма, обложить код тестами.


Что сделано:
Проект создан в Visual Studio 2017 15.7.5

В качестве структуры данных для хранения матрицы был выбран одномерный массив. Возможно, лучше было выбрать jagged array. Сначала я, возможно ошибочно, посчитал, что одномерный массив будет для целей умножения быстрее чем как multidimensional array (который в CLR по общему мнению медленный), так и jagged array.

Алгоритм для умножения матриц выбрал стандартный (алогритм Штрассена на первый взгляд был очень объемным в реализации, не говоря о каком-то наиболее современном - если правильно понял, с использованием тензоров).

С определенного размера матриц задействуется многопоточное умножение. Порог этот наверное условный. Просто подсмотрел идею, что для совсем маленьких матриц, возможно, нет смысла запускать несколько потоков, накладные расходы будут больше.

Непосредственно при умножении соответствующий столбец и строка копируются в отдельные массивы. Возможно, это позволит более оптимально использовать кэш процессора, т.к. это скорее всего будут довольно маленькие массивы, из которых происходит последовательное чтение.
